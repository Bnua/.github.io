Python中数据可视化的下一级

如何使用单行Python制作外观漂亮，完全互动的图表

威尔克尔森
1月9日 · 8 分钟阅读

沉没成本谬误是人类成为牺牲品的众多有害认知偏见之一。它指的是我们倾向于继续将时间和资源投入到失去的原因中，因为我们已经花了很多时间 - 沉没 - 花了很多时间去追求。沉没成本的谬误适用于保持工作时间比我们应该做的更长，即使很明显它也不会起作用，也会偏离一个项目，是的，继续使用一个繁琐的，过时的绘图库 - matplotlib - 当效率更高时，存在互动和更好看的替代品。

在过去的几个月里，我意识到我使用的唯一原因matplotlib是数百小时我沉浸在学习复杂的语法中。这种复杂性导致StackOverflow数小时的挫折，弄清楚如何格式化日期或添加第二个y轴。幸运的是，这是Python绘图的好时机，在探索了这些选项之后，一个明显的赢家 - 在易用性，文档和功能方面 - 是一个简洁的Python库。在本文中，我们将深入研究plotly如何在更短的时间内制作出更好的情节 - 通常只需一行代码。

本文的所有代码都可以在GitHub上找到。图表都是交互式的，可以在NBViewer上查看。

Plotly简要概述

该 plotly Python包是一个基于开源库的开源库 plotly.js 哪一个 反过来建立在 d3.js。我们将使用一个名为design的包装器来cufflinks设计与Pandas数据帧一起使用。因此，我们的整个堆栈是袖扣> plotly> plotly.js> d3.js，这意味着我们可以通过d3令人难以置信的交互式图形功能获得Python编码的效率。

（Plotly本身就是一家拥有多种产品和开源工具的图形公司.Python库可以免费使用，我们可以在离线模式下创建无限图表，在线模式下最多可以创建25个图表，以便与全世界分享。）

本文中的所有工作都是在Jupyter笔记本中完成的，其中的图形+袖扣在离线模式下运行。安装了plotly和cufflinks并pip install cufflinks plotly导入以下内容以在Jupyter中运行：

＃标准plotly进口
导入plotly.plotly为PY 
进口plotly.graph_objs如去
从plotly.offline进口IPLOT，init_notebook_mode ＃使用plotly在离线模式+袖扣
进口袖扣
cufflinks.go_offline（连接= TRUE）
init_notebook_mode（连接=真）

单变量分布：直方图和箱线图

单变量 - 单变量 - 图是开始分析的标准方法，直方图是绘制分布图的首选图（虽然它有一些问题）。在这里，使用我的中文文章统计信息（你可以看到如何在这里获取你自己的统计数据或在这里使用我的）让我们制作一个文章拍手数量的交互式直方图（df是一个标准的Pandas数据帧）：

df ['claps']。iplot（kind ='hist'，xTitle ='claps'，
                  yTitle ='count'，title ='Claps Distribution'）

对于那些习惯matplotlib，我们所要做的就是再添加一个字母（iplot而不是plot），我们得到一个更好看的交互式图表！我们可以点击数据来获取更多细节，放大图的各个部分，我们稍后会看到，选择要突出显示的不同类别。

如果我们想要绘制叠加的直方图，那就简单了：

df [['time_started'，'time_published']]。iplot（
    kind ='hist'，
    histnorm ='percent'，
    barmode ='overlay'，
    xTitle ='时间'，
    yTitle ='（％）的文章' ，
    title ='时间开始和发布时间'）

通过一点pandas操作，我们可以做一个条形图：

＃重新采样到每月频率并绘制
df2 = df [['view'，'reads'，'published_date']]。\ 
         set_index（'published_date'）。\ 
         resample（'M'）。mean（）df2.iplot（kind ='bar'，xTitle ='Date'，yTitle ='Average'，
    title ='每月平均视图和读数'）

我们看到，我们可以将大熊猫的力量与情节+袖扣结合起来。对于每个故事的出版物粉丝的箱线图，我们使用a pivot然后绘图：

df.pivot（columns ='publication'，values ='fans'）。iplot（
        kind ='box'，
        yTitle ='fans'，
        title ='Fans Distribution by Publication'）

交互性的好处是我们可以根据需要探索和分组数据。盒子图中有很多信息，如果没有查看数字的能力，我们会错过大部分信息！

散点图

散点图是大多数分析的核心。它允许我们看到变量随时间的演变或两个（或更多）变量之间的关系。

时间序列

相当一部分的真实数据有一​​个时间元素。幸运的是，情节+袖扣的设计考虑了时间序列的可视化。让我们制作一篇关于我的TDS文章的数据框，看看趋势是如何变化的。

创建数据框架走向数据科学文章
tds = df [df ['publication'] =='走向数据科学']。\ 
         set_index（'published_date'）＃绘制读取时间作为时间序列
tds [['claps'，'粉丝'，'标题']]。iplot（
    y ='claps'，mode ='lines + markers'，secondary_y ='粉丝'，
    secondary_y_title ='粉丝'，xTitle ='日期'，yTitle ='拍手'，
    文字='title'，title ='随时间变化的风扇和拍手'）

在这里，我们在一行中做了很多不同的事情：

  • 自动获得格式良好的时间序列x轴
  • 添加辅助y轴因为我们的变量具有不同的范围
  • 将文章标题添加为悬停信息

有关更多信息，我们还可以非常轻松地添加文本注释：

tds_monthly_totals.iplot（
    mode ='lines + markers + text'，
    text = text，
    y ='word_count'，
    opacity = 0.8，
    xTitle ='Date'，
    yTitle ='Word Count'，
    title =' 按月计算总字数' ）

对于由第三个分类变量着色的双变量散点图，我们使用：

    title ='阅读百分比与阅读比率按出版物“）

让我们通过使用日志轴（指定为绘图布局）（参见Plotly文档中的布局细节）以及通过数值变量调整气泡来更复杂一点：

tds.iplot（
    x ='word_count'，
    y ='reads'，
    size ='read_ratio'，
    text = text，
    mode ='markers'，
    ＃Log xaxis

        xaxis = dict（type ='log'，title） ='字数'），

有了更多的工作（详见笔记本），我们甚至可以在一个图表上放置四个变量（这不建议）！

和以前一样，我们可以将pandas与plotly + cufflinks结合起来，以获得有用的图表

df.pivot_table（

有关添加功能的更多示例，请参阅笔记本或文档。我们可以使用单行代码在文本中添加文本注释，参考线和最佳拟合线，并且仍然可以进行所有交互。

Advanced Plots

Now we’ll get into a few plots that you probably won’t use all that often, but which can be quite impressive. We’ll use the plotly figure_factory, to keep even these incredible plots to one line.

Scatter Matrix

When we want to explore relationships among many variables, a scattermatrix (also called a splom) is a great option:

import plotly.figure_factory as fffigure = ff.create_scatterplotmatrix(
    df[['claps', 'publication', 'views',      
        'read_ratio','word_count']],
    diag='histogram',
    index='publication')

Even this plot is completely interactive allowing us to explore the data.

Correlation Heatmap

To visualize the correlations between numeric variables, we calculate the correlations and then make an annotated heatmap:

corrs = df.corr()figure = ff.create_annotated_heatmap(
    z=corrs.values,
    x=list(corrs.columns),
    y=list(corrs.index),
    annotation_text=corrs.round(2).values,
    showscale=True)

Editing in Plotly Chart Studio

When you make these plots in the notebook, you’ll notice a small link on the lower right-hand side on the graph that says “Export to plot.ly”. If you click that link, you are then taken to the chart studio where you can touch up your plot for a final presentation. You can add annotations, specify the colors, and generally clean everything up for a great figure. Then, you can publish your figure online so anyone can find it with the link.

Conclusions

The worst part about the sunk cost fallacy is you only realize how much time you’ve wasted after you’ve quit the endeavor. Fortunately, now that I’ve made the mistake of sticking with matploblib for too long, you don’t have to!

When thinking about plotting libraries, there are a few things we want:

  1. One-line charts for rapid exploration
  2. Interactive elements for subsetting/investigating data
  3. Option to dig into details as needed
  4. Easy customization for final presentation

截至目前，在Python中完成所有这些操作的最佳选择是情节。Plotly允许我们快速进行可视化，并帮助我们通过交互性更好地洞察我们的数据。此外，让我们承认，绘图应该是数据科学中最令人愉快的部分之一！与其他图书馆一起，情节变成了一项繁琐的任务，但是在情节上，制作一个伟大的人物再次感到高兴！

现在是2019年，现在是时候升级您的Python绘图库，以便在数据科学可视化中实现更高的效率，功能和美学。
